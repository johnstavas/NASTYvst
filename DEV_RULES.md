# Shags VST — Plugin System Development Rules

This document is the permanent rulebook for every plugin in this codebase.
Rules marked **mandatory** must be followed unless the plugin header explicitly
declares an exemption with a reason. Rules marked **recommended** may be
violated if doing so serves the plugin's identity, but the violation must be
annotated.

Treat this file as contract. If you change the rules, change this file in the
same commit, and explain what changed and why in `RULES_CHANGELOG` at the
bottom.

---

## A. State + Defaults

- **A1 · single-defaults-table** *(mandatory)* — One `DEFAULTS =
  Object.freeze({...})` per plugin. Knob `defaultVal`, `useState` init, save
  payload, and double-click reset all resolve through it.
- **A2 · persisted-keys-derived** *(mandatory)* — `PERSISTED_KEYS =
  Object.keys(DEFAULTS)`. Save payload is generated by iterating that list.
- **A3 · legacy-migration-explicit** *(mandatory)* — Any change to a persisted
  default requires an explicit migration shim naming the old value. Never
  rewrite saves silently.

## B. DSP + Signal Path

- **B1 · rack-contract-exact** *(mandatory)* — `{ input, output, chainOutput?,
  setIn, setOut, setMix, setBypass, isBypassed, applyBulk, dispose, ... }`.
- **B2 · no-orphan-nodes** *(mandatory, lintable)* — Every `ctx.create*` is
  connected or deleted. No "kept for later" nodes.
- **B3 · in-trim-pre-split** *(mandatory)* — IN trim applied before the dry/wet
  split so both legs scale equally.
- **B4 · no-dual-writers** *(mandatory)* — Modulators and user knobs never
  directly write the same AudioParam. Use a series modulation node instead.
- **B5 · composer-slots-for-shared-targets** *(mandatory)* — Multi-knob targets
  use per-knob slots and a `recomputeX()` combiner.

## C. Mix / Phase / Latency

- **C1 · equal-power-drywet** *(mandatory)* — MIX uses `cos(m·π/2)` /
  `sin(m·π/2)`.
- **C2 · linear-parallel-correlated** *(mandatory)* — Parallel correlated legs
  use linear crossfade.
- **C3 · dry-matches-wet-latency** *(mandatory)* — Dry leg compensated to match
  wet-chain latency (delay baselines + comp lookahead + oversample AA).
- **C4 · widener-post-merge** *(mandatory)* — Widener sits after dry/wet merge.
- **C5 · width-zero-is-true-mono** *(mandatory)* — Width=0 ⇒ both outputs =
  `(L+R)/2` at matched level.
- **C6 · oversample-accounted** *(mandatory)* — Oversample latency covered, or
  oversample is `'none'`.
- **C7 · modulated-delay-baseline** *(mandatory)* — Baseline ≥ worst-case
  negative LFO swing + 50 % headroom.

## D. Presets + Recall

- **D1 · presets-are-full-snapshots** *(mandatory)* — Presets write every
  parameter that defines their voice.
- **D2 · setCharacter-all-or-nothing** *(mandatory)* — Engine `setCharacter`
  applies the full preset, or is removed from the public API.
- **D3 · applyBulk-exists** *(mandatory)* — Every engine exposes
  `applyBulk(obj)`. Unknown keys ignored.
- **D4 · preset-highlight-scoped-clear** *(mandatory)* — Highlight clears only
  for knobs the preset explicitly wrote.
- **D5 · preset-suppression-gate** *(mandatory)* — `applyingPresetRef` flag
  gates per-knob effects during bulk writes.

## E. UI / UX

- **E1 · no-inline-components** *(mandatory, lintable)* — No subcomponents
  defined inside another component's render body.
- **E2 · fmt-is-rendered** *(mandatory)* — If `fmt` is a knob prop, it appears
  on screen.
- **E3 · tooltip-on-every-control** *(mandatory)* — Every interactive element
  has a `title`.
- **E4 · defaultVal-references-DEFAULTS** *(mandatory, lintable)* — Knob
  `defaultVal` must be `DEFAULTS.<key>`, never a literal.
- **E5 · visual-disable-matches-DSP-disable** *(mandatory)* — Bypassed stages
  visually dim their controls and set `pointer-events: none`.

## F. Metering

- **F1 · meters-show-true-fs** *(mandatory)* — Linear 0..1 = 0..FS. No visual
  gain.
- **F2 · zero-db-marker** *(mandatory)* — Visible 0 dB reference on every
  meter.
- **F3 · input-and-output-both** *(mandatory)* — Both IN and OUT meters always
  visible.
- **F4 · raf-change-gated** *(mandatory)* — Meter RAF updates skip setState
  when value hasn't changed beyond epsilon.

## G. Architecture

- **G1 · engine-view-split** *(mandatory)* — Engines have zero React; views
  have zero raw DSP internals.
- **G2 · one-engine-ref** *(mandatory)* — One `engineRef` per instance, created
  in `useEffect`, disposed on unmount.
- **G3 · shared-context** *(mandatory)* — Plugins receive `sharedSource`; never
  create their own AudioContext.
- **G4 · registry-membership** *(mandatory)* — Shippable plugins live in the
  migration registry with at least a `legacy` variant.
- **G5 · menu-wiring-complete** *(mandatory, lintable)* — A plugin is NOT
  shippable until all FOUR wiring sites are touched in the same commit:
    1. `src/main.jsx` — `import XxxOrb from './.../XxxOrb.jsx';`
    2. `src/main.jsx` — entry added to `PLUGIN_CATEGORIES[<Category>].items`
       as `['typeString', 'Display Label']`
    3. `src/main.jsx` — render switch: `inst.type === 'typeString' ? <XxxOrb ... />`
    4. `src/migration/registry.js` — `REGISTRY` entry with `legacyType`
       matching the `typeString` from (2)
  The migration registry alone is NOT enough. It only feeds the QC-mode
  tabs. The default rack menu is the hardcoded `PLUGIN_CATEGORIES` array
  in `main.jsx`, and the render tree is a hardcoded `inst.type === …`
  switch. Skip any of these four and the plugin appears to compile but
  will never appear in the menu, or will appear but mount as nothing.
  Missed this on MANchild 2026-04-18 — hence this rule.

## H. Lifecycle / Cleanup

- **H1 · dispose-stops-everything** *(mandatory)* — Every Oscillator, every
  BufferSource, every RAF, every setTimeout is cleared. `input`/`output`
  disconnected.
- **H2 · silent-fade-in-on-mount** *(mandatory)* — ~180 ms silent hold + ~60 ms
  ramp to unity on engine construct.
- **H3 · applyingPreset-microtask-release** *(mandatory)* — Suppression flag
  releases via `queueMicrotask`.

## I. Documentation / Comments

- **I1 · topology-header-matches-graph** *(mandatory)* — Topology comment at
  top of engine matches the built graph. Update in the same commit as any
  graph change.
- **I2 · static-trim-rationale** *(mandatory)* — Every static gain/trim
  value carries a comment explaining what it was tuned for.
- **I3 · bypass-behavior-labeled** *(mandatory)* — Engine header declares
  bypass behavior: `zero-latency` | `phase-safe (N ms latency)` | `true bypass
  (separate path)`.
- **I4 · reported-latency** *(mandatory)* — `getLatency()` returns plugin
  latency in seconds.

## J. Chassis / Layout Standards

- **J1 · chassis-380x500** *(mandatory unless exempted)* — Outer 380 × 500,
  border-radius 5.
- **J2 · inner-art-380x280** *(mandatory unless exempted)* — Inner art 380 ×
  280. Exemption must appear in the plugin header.
- **J3 · io-strip-at-bottom** *(recommended)* — Bottom strip carries IN meter,
  OUT meter, IN/OUT/MIX knobs.
- **J4 · header-has-bypass-and-name** *(mandatory)* — Name matches registry
  `displayLabel`, chain pill, and menu entry.

# Future Plugin Preflight Checklist

A new plugin is **not shippable** until every box below is checked.

## State
- [ ] One `DEFAULTS = Object.freeze({...})` table exists
- [ ] Every knob `defaultVal` references `DEFAULTS.<key>`
- [ ] Every `useState` initializer reads `init.<key> ?? DEFAULTS.<key>`
- [ ] `PERSISTED_KEYS = Object.keys(DEFAULTS)` drives the save payload
- [ ] Legacy save migrations (if any) are commented with old values

## DSP / Signal
- [ ] Engine returns the full rack contract (`input, output, chainOutput?,
      setIn, setOut, setMix, setBypass, isBypassed, applyBulk, dispose`)
- [ ] No orphan nodes — every `ctx.create*` is connected
- [ ] IN trim applied before the dry/wet split
- [ ] No two code paths write the same AudioParam
- [ ] Multi-knob shared targets use composer slots

## Mix / Phase / Latency
- [ ] MIX uses equal-power crossfade
- [ ] Correlated parallel legs use linear crossfade
- [ ] Dry leg latency-matched to wet chain
- [ ] Widener sits after dry/wet merge
- [ ] Width=0 produces `(L+R)/2` on both channels
- [ ] WaveShaper oversample latency is either compensated or set to `'none'`
- [ ] Modulated delays have baseline ≥ worst-case negative swing + 50 %

## Presets
- [ ] Presets are full-state snapshots
- [ ] Engine `setCharacter` either applies every preset field or is not
      exposed on the public API
- [ ] `applyBulk(obj)` present and ignores unknown keys
- [ ] Preset highlight scoped-clears only for keys the preset wrote
- [ ] `applyingPresetRef` gate wraps the bulk write

## UI
- [ ] No component defined inside another component's render body
- [ ] Every knob renders its `fmt` value
- [ ] Every interactive element has a `title` tooltip
- [ ] Knob `defaultVal` = `DEFAULTS.<key>`
- [ ] Bypassed stages visually dim + `pointer-events: none`

## Metering
- [ ] IN and OUT meters both present
- [ ] Linear 0..1 = 0..FS — no visual gain
- [ ] Visible 0 dB reference tick / color transition
- [ ] RAF meter updates are change-gated

## Architecture
- [ ] Engine contains zero React
- [ ] View contains zero raw DSP internals
- [ ] One `engineRef` via `useRef`, created in `useEffect`
- [ ] Uses `sharedSource.ctx` — no self-created AudioContext
- [ ] Registered in `src/migration/registry.js` with a `legacy` variant
- [ ] **Menu wiring (G5) — all four sites touched in the same commit:**
  - [ ] `src/main.jsx` — `import XxxOrb` line added
  - [ ] `src/main.jsx` — entry in `PLUGIN_CATEGORIES[<cat>].items`
  - [ ] `src/main.jsx` — render branch in the `inst.type === '…' ?` switch
  - [ ] `src/migration/registry.js` — `REGISTRY` entry with `legacyType`
        equal to the `typeString` used in `PLUGIN_CATEGORIES`
- [ ] Load the app, open the Add menu, confirm the plugin appears AND mounts
      without console errors. (A clean `vite build` is not enough — the
      menu and switch are runtime-only.)

## Lifecycle
- [ ] `dispose()` stops every Oscillator, BufferSource, RAF, and setTimeout
- [ ] Silent fade-in on mount (~180 ms hold + ~60 ms ramp)
- [ ] `applyingPresetRef` releases in `queueMicrotask`

## Documentation
- [ ] Engine header topology comment matches built graph
- [ ] Static trims annotated with rationale
- [ ] Bypass labeled (`zero-latency` | `phase-safe (N ms)` | `true bypass`)
- [ ] `getLatency()` returns seconds

## Chassis
- [ ] Outer 380 × 500, radius 5 — or exemption declared in header
- [ ] Inner art 380 × 280 — or exemption declared in header
- [ ] Name consistent across registry, chain pill, menu, header, file names
